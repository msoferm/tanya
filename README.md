# מערכת ניהול לידים — אקדמיה לתניא

מערכת לניהול לידים עם 3 סוגי משתמשים, חלוקה אוטומטית בין טלפנים, תזכורות,
סנכרון עם WooCommerce וזיהוי מקור תנועה (UTM) מטופסי אלמנטור.

צבעי המערכת: שחור / לבן / ירוק `rgb(7,153,101)`.

---

## 🌐 האתר החי

**https://tanya-cc32b.web.app**

| רכיב | פלטפורמה |
|------|-----------|
| ממשק (Frontend) | Firebase Hosting |
| שרת ה-API (Backend) | Firebase Cloud Functions — פונקציה `api` |
| מסד נתונים | Supabase (PostgreSQL) |
| קוד מקור | https://github.com/msoferm/tanya |

### עדכון גרסה (פריסה מחדש)

לאחר שינוי בקוד, מהתיקייה הראשית:

```bash
firebase deploy            # פורס Frontend + Backend
firebase deploy --only hosting     # רק הממשק
firebase deploy --only functions   # רק השרת
```

> דרוש קובץ `backend/.env` מקומי (מכיל את חיבור ה-Supabase והסודות) —
> הוא לא נשמר ב-git. אם משכתם את הקוד מ-GitHub, צרו אותו מתוך `backend/.env.example`.

| תפקיד | רואה | מה מותר לו |
|--------|------|-------------|
| **טלפן** (`telephonist`) | רק את הלידים שמשויכים אליו | עריכת לידים, תזכורות, חיפוש, וואטסאפ |

## 3 חלקי המערכת

### חלק 1 — הטלפנים
- **מניעת כפילויות** — לפי טלפון (מנורמל) ומייל, אוטומטית בכל ערוץ קליטה.
- **דף התחברות אישי** — כל טלפן רואה רק את הלידים שלו, ללא גישה לדפים אחרים.
- **חלוקה שווה** — כל ליד חדש משויך אוטומטית לטלפן עם הכי מעט לידים שהתקבלו היום
  (20 לידים + 4 טלפנים = 5 לכל אחד).
- **שדות הליד** — שם, סטטוס, טלפון, מייל, הערות חופשיות, מקור, יום פתוח,
  וובינר + משך בדקות, מתי נרשם, עדכון אחרון.
- **סטטוסים חופשיים** — יצירה ועריכה של סטטוסי לקוח עם צבעים.
- **סרגל סינון + עריכה מרובה** — בחירה של כמה לידים ושינוי סטטוס/שיוך בבת אחת.
- **תזכורות** — קביעת מועד חזרה לליד; דף תזכורות מסודר לפי יום ("היום" / הקרובים).
- **חיפוש** — לפי שם, מייל או טלפון, כולל **חצי מספר טלפון**.
- **וואטסאפ** — לחיצה על הכפתור פותחת את WhatsApp עם המספר של הליד.

### חלק 2 — חיבור לאתר
- **זיהוי רכישות** — וובהוק/סנכרון מ-WooCommerce מסמן אוטומטית ליד שרכש.
- **יצירת חשבון** — כפתור בכרטיס הליד יוצר משתמש באתר ה-WordPress
  ושולח אליו פרטי התחברות במייל, לכניסה ישירה לקורס.

### חלק 3 — משתמש על
- **ייבוא אקסל** ללא דריסת נתונים קיימים, עם זיהוי כפילויות (בקובץ ומול המערכת).
- **דף ייבוא מסודר** עם תצוגה מקדימה — מציג אילו שורות ייובאו, מה ידולג ולמה.
- **מיפוי עמודות** — בחירה איזו עמודה בקובץ ממופה לאיזה שדה.
- **ייצוא אקסל** עם בחירת השדות והסינונים לייצוא.

## התקנה והרצה

### דרישות מקדימות
- Node.js 18+ ו-npm
- PostgreSQL 14+ (או Docker)

### 1. מסד הנתונים

עם Docker (הכי פשוט) — מהתיקייה הראשית:
```bash
docker compose up -d
```

או PostgreSQL מותקן מקומית — צרו מסד נתונים ומשתמש שתואמים ל-`backend/.env`.

### 2. השרת (Backend)
```bash
cd backend
npm install
# הגדרות נמצאות בקובץ .env (כבר קיים עם ערכי ברירת מחדל לפיתוח)
npm run db:init      # יצירת הטבלאות + נתוני התחלה
npm run dev          # מריץ על http://localhost:5001
```

### 3. הממשק (Frontend)
```bash
cd frontend
npm install
npm run dev          # מריץ על http://localhost:5174
```

פתחו את הדפדפן בכתובת **http://localhost:5174**.

> **פורטים:** השרת על 5001, הממשק על 5174, מסד הנתונים על 5433 —
> נבחרו כך כדי לא להתנגש עם פרויקט ה-CRM הקיים. ניתן לשנות בקובץ `.env`
> ו-`docker-compose.yml`.

### פרטי כניסה ראשוניים

| תפקיד | מייל | סיסמה |
|--------|------|--------|
| משתמש על | `admin@tanya.local` | `Admin123!` |
| טלפנים | `agent1@tanya.local` … `agent4@tanya.local` | `Tanya123!` |

> ⚠️ החליפו את הסיסמאות מיד אחרי הכניסה הראשונה (מסך "החשבון שלי").

## חיבור טופס אלמנטור

המערכת קולטת לידים ישירות מטופס Elementor Pro וגם מזהה את מקור התנועה (UTM).

1. **כתובת ה-Webhook** מופיעה במסך *הגדרות ואינטגרציות* (מועתקת בלחיצה). למשל:
   `https://SERVER/api/webhooks/elementor?token=...`
2. בעורך הטופס באלמנטור → *Actions After Submit* → הוסיפו **Webhook**
   והדביקו את הכתובת.
3. **שדות הטופס** — שמות השדות (Field ID) שהמערכת מזהה: `name`, `phone`, `email`.
4. **קליטת UTM** — הוסיפו לטופס שדות מוסתרים (Hidden) עם ה-IDs:
   `utm_source`, `utm_medium`, `utm_campaign`, `utm_term`, `utm_content`.
   לכל שדה כזה, ב-*Advanced → Default Value*, בחרו מקור **Get** והכניסו את שם
   הפרמטר (למשל `utm_source`). כך הערך נשאב מכתובת ה-URL ונשלח עם הליד.

ליד שמגיע מהטופס משויך אוטומטית לטלפן הבא בתור (חלוקה שווה).

## חיבור WooCommerce

1. ב-WordPress: *WooCommerce → Settings → Advanced → REST API → Add key*
   (הרשאות Read/Write). העתיקו את ה-Consumer key וה-Consumer secret.
2. מלאו ב-`backend/.env`:
   ```
   WC_URL=https://your-academy-site.co.il
   WC_CONSUMER_KEY=ck_...
   WC_CONSUMER_SECRET=cs_...
   ```
   והפעילו מחדש את השרת.
3. **זיהוי רכישות אוטומטי** — ב-WooCommerce *Settings → Advanced → Webhooks*
   הוסיפו Webhook על האירוע **Order created** (וגם **Order updated**) עם
   כתובת ה-Delivery URL ממסך ההגדרות שלנו. ליד עם מייל/טלפון תואם יסומן כ"רכש".
4. **סנכרון יזום** — במסך ההגדרות יש כפתור "סנכרן עכשיו" שמושך הזמנות אחרונות.
5. **יצירת חשבון** — בכרטיס ליד (משתמש על) כפתור "צור חשבון ושלח פרטים"
   פותח משתמש באתר ושולח את פרטי ההתחברות במייל.

## שליחת מיילים

לשליחת פרטי התחברות ללקוחות, מלאו ב-`backend/.env` את פרטי ה-SMTP
(`SMTP_HOST`, `SMTP_USER`, `SMTP_PASS` וכו'). ללא הגדרה זו המערכת תפעל,
אך מיילים לא יישלחו (WooCommerce עדיין יכול לשלוח מייל חשבון משלו).
